# Архитектура

Система построена как оркестратор с API-first логикой и fallback в ручной режим.

## Контуры работы

- API-first: после выбора событий и списка СМИ запускается `auto-run` и шаги 2–5 выполняются автоматически.
- Manual fallback: любой шаг можно выполнить через copy/paste и `ingest-response`.

## Шаги

1. `step1` — поиск кандидатов событий (OpenAI).
2. `step2` — анализ повестки и сценариев (OpenAI).
3. `step3` — профиль редакционной логики СМИ (OpenAI).
4. `step4` — генерация заголовков и лидов (OpenAI + DeepSeek + Qwen).
5. `step5` — внутренняя редакционная комиссия (OpenAI).
6. Ручная экспертная оценка и финальный выбор.

## Сохранение артефактов

Для каждого шага сохраняются:

- отправленный prompt;
- raw-ответ;
- parsed JSON;
- summary / comparison;
- manifest сессии.

При `auto-run` артефакты пишутся по run-id:

- `step2/runs/<event_id>/`
- `step3/runs/<event_id>__<outlet>/`
- `step4/runs/<event_id>__<outlet>/`
- `step5/runs/<event_id>__<outlet>/`

## Интеграция

CLI-ядро может использоваться как backend для:

- мессенджер-бота;
- web/UI-оболочки;
- корпоративной ИС с внешним триггером запуска пайплайна.
