# Архитектура

Система построена как пошаговый оркестратор с ручным циклом `copy -> paste -> get reply -> paste back -> continue`.

## Слои

- `cli.py` — входная точка и команды пользователя.
- `steps/` — логика подготовки каждого этапа пайплайна.
- `providers/` — API-ready обёртки провайдеров (OpenAI, DeepSeek, Qwen) и ручной провайдер.
- `storage.py` и `session.py` — хранение сессий, артефактов, манифеста.
- `parser.py` и `formatter.py` — разбор ответов моделей и человекочитаемые summary.
- `compare_outputs.py` — сводный сравнительный блок по шагу 4.

## Режимы

- `manual` (MVP): система готовит промпт, сохраняет артефакты и принимает вставленный ответ.
- `api`: при наличии ключей и SDK может выполнить вызов модели напрямую; при отсутствии ключей автоматически предлагает manual-flow.

## Принцип сохранения

Для каждого шага сохраняются:

- отправленный промпт;
- raw-ответ;
- parsed JSON;
- summary/сравнение;
- обновлённый `manifest.json` сессии.
